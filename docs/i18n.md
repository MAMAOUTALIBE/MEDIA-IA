# Internationalisation — plan d'adoption

## État actuel

Les libellés sont en français en dur dans le code (`apps/web/src/components/**/*.tsx`). Une migration vers un système i18n est planifiée mais non encore implémentée — l'extraction sans casser visuellement nécessite un refactor coordonné de ~60 fichiers.

## Lib retenue

**`next-intl`** — la lib de référence pour Next.js App Router. Server-component-first, type-safe, supporte ICU MessageFormat.

## Plan d'adoption (à exécuter quand prioritaire)

### Étape 1 — Setup

```bash
pnpm --filter web add next-intl
```

Créer `apps/web/messages/fr.json` avec les libellés.
Créer `apps/web/src/i18n/routing.ts` et `apps/web/src/i18n/request.ts`.
Ajouter `next-intl/plugin` dans `next.config.ts`.

### Étape 2 — Route group `[locale]`

Déplacer toutes les routes sous `src/app/[locale]/` :
- `src/app/[locale]/(marketing)/page.tsx`
- `src/app/[locale]/dashboard/*`
- `src/app/[locale]/mobile/*`

Middleware `src/middleware.ts` :
```ts
import createMiddleware from 'next-intl/middleware';
import { routing } from './i18n/routing';
export default createMiddleware(routing);
export const config = {
  matcher: ['/((?!api|_next|_vercel|.*\\..*).*)']
};
```

### Étape 3 — Extraire les chaînes

Pour chaque page/composant :
- Remplacer `"Tableau de bord"` par `t("dashboard.home.title")`
- Importer `useTranslations` (client) ou `getTranslations` (server)

Convention de namespace :
- `common.*` — libellés génériques (boutons, statuts)
- `nav.*` — sidebar et navigation
- `dashboard.home.*` — page d'accueil
- `dashboard.contenus.*` — module contenus
- `dashboard.ai.*` — assistant IA
- etc.

### Étape 4 — Locales additionnelles

Démarrer avec **`fr` puis `en`**. Anglais minimum pour les démos internationales.
Plus tard : wolof (`wo`), arabe (`ar` avec RTL via `dir="rtl"`), portugais (`pt`).

### Étape 5 — Translation Management

À 500+ clés : adopter **Lokalise** ou **Crowdin** pour piloter la traduction via une équipe externalisée.

## Estimation effort

- Setup + middleware : 1 j
- Extraction `(marketing)/page.tsx` + sidebar : 1 j
- Extraction `dashboard/*` (10 pages) : 3 j
- Extraction `mobile/*` (3 pages) : 1 j
- Extraction composants partagés (AI Assistant, Notifications, etc.) : 1 j
- Tests Playwright multi-locales : 1 j

**Total ≈ 1.5 semaine** pour atteindre FR + EN sur tout le produit.

## Pourquoi pas tout de suite

Aujourd'hui :
- 1 seul locale (FR) à servir
- Pas de demande client EN
- Phase 0/1 (backend) prioritaire

À déclencher quand : démo internationale planifiée OU décision produit de cibler plusieurs marchés.
